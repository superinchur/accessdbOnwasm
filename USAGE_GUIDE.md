# AccessDB Browser (accessdb-on-wasm) 사용 가이드

이 프로젝트는 웹 브라우저(`Tauri` 기반 데스크톱 앱)에서 Microsoft Access(`.mdb`, `.accdb`) 파일을 열고 SQL 쿼리를 실행할 수 있게 해주는 도구입니다.

---

## 1. 개요 및 아키텍처

이 프로그램은 크게 두 부분으로 나뉩니다.
- **Frontend (Tauri/React)**: 사용자 인터페이스(UI). SQL 에디터와 결과를 보여주는 테이블이 포함되어 있습니다.
- **Bridge (Bun HTTP Sidecar)**: 실제 Access 파일을 읽어서 SQLite 메모리 DB로 변환해주는 백엔드 서버입니다. Bun 런타임을 사용하며, `mdb-reader` 라이브러리를 통해 Access 데이터를 추출합니다.

---

## 2. 사전 준비 사항

프로그램을 실행하거나 빌드하기 위해 다음 소프트웨어가 설치되어 있어야 합니다.
- **Node.js** (v18 이상 권장)
- **Bun** (Bridge 서버 빌드 및 실행을 위해 필요)
- **Rust/Tauri CLI** (데스크톱 앱 실행을 위해 필요)

---

## 3. 설치 및 실행 방법 (개발자 모드)

### (1) 의존성 설치
터미널에서 프로젝트 루트 디렉토리로 이동한 후 다음 명령어를 실행합니다.
```bash
npm install
```

### (2) 개발 모드 실행
Vite 개발 서버와 Tauri 앱을 동시에 실행합니다.
```bash
npm run tauri dev
```
앱이 실행되면 GUI 창이 나타납니다.

---

## 4. 주요 기능 및 사용법

### (1) 데이터베이스 파일 로드
- 앱 실행 후 `Open Access File` 버튼(또는 유사한 아이콘)을 클릭하여 로컬에 있는 `.accdb` 또는 `.mdb` 파일을 선택합니다.
- 파일이 로드되면 Bridge 서버가 데이터를 읽어 메모리에 SQL 환경을 구축합니다.

### (2) 테이블 목록 확인
- 왼쪽 사이드바나 별도의 목록에서 로드된 데이터베이스의 테이블 이름들을 확인할 수 있습니다.

### (3) SQL 쿼리 실행
- 중앙의 SQL 에디터(CodeMirror 기반)에 실행하고 싶은 SQL 쿼리를 입력합니다.
  - 예: `SELECT * FROM 테이블이름 LIMIT 10`
- `Run` 버튼을 누르면 하단 결과 창에 데이터가 그리드 형태로 표시됩니다.

### (4) 데이터 결과 내보내기
- 쿼리 결과를 CSV 파일 등으로 내보낼 수 있는 기능이 포함되어 있습니다. (프로그램 상의 `Export` 버튼 활용)

---

## 5. 빌드 및 배포

실제 실행 파일(`.exe`)을 만들고 싶을 때 수행합니다.

### (1) Bridge 바이너리 빌드
Bun을 사용하여 백엔드 브릿지 서버를 실행 파일로 컴파일합니다.
```bash
npm run build:bridge
```

### (2) Tauri 앱 빌드
최종 설치 파일을 생성합니다.
```bash
npm run tauri:build
```
빌드가 완료되면 `src-tauri/target/release/bundle` 폴더 내에 설치 파일이 생성됩니다.

---

## 6. 테스트 실행 방법

프로젝트에는 백엔드 브릿지 서버의 동작을 검증하기 위한 테스트 코드가 포함되어 있습니다.

### (1) 브릿지 서버 테스트 실행
Bun의 테스트 러너를 사용하여 API 엔드포인트와 인증 로직을 테스트합니다.
```bash
bun test src-bridge/server.test.ts
```
이 테스트는 `/health`, `/tables`, `/query` 등 주요 API가 정상적으로 동작하는지 확인합니다.

### (2) 테스트용 데이터 파일
루트 디렉토리에 있는 다음 파일들은 테스트 및 샘플용 데이터베이스 파일입니다.
- `test.accdb`: Access 2007+ 형식 핵심 데이터베이스 샘플.
- `test.mdb`: 레거시 Access 형식 샘플.
이 파일들을 앱에서 직접 열어서 쿼리 실행을 테스트해 볼 수 있습니다.

---

## 7. 주의 사항 및 한계

- **읽기 전용**: 현재 버전은 Access 파일에서 데이터를 읽고 쿼리하는 용도이며, 원본 파일을 수정(INSERT/UPDATE/DELETE)하여 저장하는 기능은 제한적일 수 있습니다.
- **데이터 타입**: 모든 데이터는 SQLite에서 `TEXT` 형식으로 처리되어 표시용으로 최적화되어 있습니다.
- **메모리 사용**: 파일을 로드할 때 전체 데이터를 메모리에 올리므로, 매우 큰 데이터베이스 파일의 경우 메모리 점유율이 높아질 수 있습니다.
